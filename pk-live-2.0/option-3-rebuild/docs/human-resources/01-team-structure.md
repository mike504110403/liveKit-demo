## 關鍵角色職責（選項3 特有）

### 資深產品經理（最關鍵）

| 職責 | 具體工作 |
|------|---------|
| **需求重整** | 從 1.0 功能清單中提煉核心需求 |
| **功能優先級** | 定義 MVP 範圍，延後非核心功能 |
| **競品分析** | 對標抖音，確定功能完整度 |
| **用戶調研** | 收集 1.0 用戶反饋，改進 2.0 |
| **需求文檔** | 產出詳細 PRD（Product Requirements Document）|

### 後端 Tech Lead

| 職責 | 具體工作 |
|------|---------|
| 架構設計 | 從零設計微服務架構 |
| 技術選型 | Go 框架、數據庫、消息隊列 |
| API 設計 | RESTful API 規範 |
| Code Review | 確保代碼質量 |

### 客戶端 Team Lead

| 職責 | 具體工作 |
|------|---------|
| 架構設計 | MVVM 架構、網絡層、狀態管理 |
| 組件設計 | 可復用組件庫 |
| 性能優化 | 流暢度、內存優化 |
| 團隊協作 | 與後端 API 聯調 |
